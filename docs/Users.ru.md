# Руководство пользователя — Users

Модуль **Users** отвечает за учётные записи в `osysHome`: кто может входить в систему, какие права получает пользователь и какие параметры профиля для него настроены.

> [!IMPORTANT]
> Создание пользователей, смена паролей и назначение роли **Администратор** стоит доверять только проверенным администраторам.

## Назначение модуля

Через `Users` администратор управляет учётными записями:

- создаёт новых пользователей;
- назначает им роль;
- настраивает стартовую страницу после входа;
- выбирает часовой пояс;
- задаёт или меняет `API key`;
- загружает аватар;
- устанавливает пароль;
- удаляет больше не нужные учётные записи.

По сути модуль работает как административный редактор объектов класса `Users`.

## Где находится модуль

Откройте верхнее меню **Администрирование** и выберите пункт **Users**.

## Что видно на главной странице

В основной таблице обычно отображаются:

- аватар и имя пользователя;
- роль;
- домашняя страница;
- время последнего входа;
- кнопки **Edit**, **Set password** и **Delete**.

Кнопка **Add user** открывает форму создания новой учётной записи.

## Основные операции

### Создать пользователя

1. Нажмите **Add user**.
2. Укажите **Username**.
3. Выберите **Role**.
4. При необходимости заполните **Home page**, **Timezone** и **API key**.
5. Нажмите **Submit**.
6. После создания откройте пользователя и выполните **Set password**.

> [!TIP]
> Если `API key` нужен для интеграций, его можно не вводить вручную, а сгенерировать позже из формы пользователя.

### Изменить профиль пользователя

1. Нажмите **Edit** напротив нужной записи.
2. Обновите поля профиля.
3. Нажмите **Submit**, чтобы сохранить изменения.

Обычно здесь меняют:

- роль;
- домашнюю страницу;
- часовой пояс;
- `API key`;
- аватар.

> [!NOTE]
> Имя пользователя стоит воспринимать как постоянный идентификатор учётной записи. Для полноценного переименования безопаснее создать новую запись и перенести доступ вручную.

### Загрузить или заменить аватар

1. Откройте пользователя через **Edit**.
2. Нажмите кнопку загрузки на карточке аватара.
3. Выберите файл изображения.
4. Подтвердите загрузку.

После успешной загрузки новый аватар сразу отображается в карточке пользователя и в списке пользователей.

### Сгенерировать новый API key

1. Откройте форму пользователя через **Edit**.
2. Найдите поле **API key**.
3. Нажмите **Generate key**.
4. Сохраните изменения кнопкой **Submit**.

Это удобно, когда ключ нужно выдать внешнему скрипту или заменить после подозрения на компрометацию.

### Установить или сменить пароль

1. Нажмите **Set password** у выбранного пользователя.
2. Заполните поля **New password** и **Repeat password** одинаковым значением.
3. Нажмите **Submit**.

Если значения не совпадают, пароль не будет сохранён.

> [!NOTE]
> Пароль не задаётся прямо в форме создания пользователя. После создания учётной записи его нужно установить отдельным действием.

### Удалить пользователя

1. Нажмите **Delete** в строке нужного пользователя.
2. Подтвердите удаление.

> [!WARNING]
> Удаление учётной записи необратимо. Если доступ нужно ограничить временно, безопаснее сначала изменить роль пользователя на менее привилегированную.

## Роли доступа

| Роль | Назначение |
| --- | --- |
| **Guest** | Минимальный доступ, обычно только просмотр интерфейса и данных. |
| **User** | Базовый рабочий доступ для повседневного использования системы. |
| **Editor** | Доступ к изменению данных там, где это разрешено правами. |
| **Administrator** | Полный административный доступ, включая управление пользователями. |

> [!IMPORTANT]
> Роль **Administrator** даёт полный контроль над системой. Назначайте её только тем, кому действительно нужен административный доступ.

## Поля профиля

### Username

Уникальное имя учётной записи. Его используют для входа и идентификации пользователя в системе.

### Role

Определяет уровень доступа пользователя к разделам системы и административным действиям.

### Home page

Страница, которая открывается после входа в систему. Обычно указывают путь вроде `/` или `/admin`.

### Timezone

Часовой пояс пользователя. От него зависит, как в интерфейсе будут отображаться время событий, журналов и последнего входа.

### API key

Секретный ключ для внешних интеграций, запросов из скриптов и других автоматизаций.

Рекомендации по работе с ключом:

- не передавайте его в открытых сообщениях;
- меняйте ключ при подозрении на утечку;
- храните ключ только в тех интеграциях, которым он действительно нужен.

### Avatar

Изображение профиля пользователя. Оно показывается в карточке редактирования и в списке пользователей.

## Типовые сценарии

### Выдать доступ новому сотруднику

1. Создайте пользователя через **Add user**.
2. Назначьте минимально необходимую роль.
3. Укажите домашнюю страницу и часовой пояс.
4. При необходимости сгенерируйте `API key`.
5. Установите пароль через **Set password**.
6. Передайте пользователю его логин и временные реквизиты доступа.

### Сбросить пароль

1. Найдите нужного пользователя в списке.
2. Откройте действие **Set password**.
3. Введите новый пароль дважды.
4. Сохраните изменения и передайте новые данные безопасным способом.

### Ограничить права

1. Откройте пользователя через **Edit**.
2. Снизьте роль до подходящего уровня.
3. Сохраните изменения.
4. Проверьте, не остались ли у пользователя старые интеграции с действующим `API key`.

### Отозвать доступ полностью

1. Убедитесь, что учётная запись больше не нужна.
2. При необходимости сохраните связанные с ней данные для аудита.
3. Удалите пользователя через **Delete**.

## Чек-лист администратора

- [ ] Для каждого пользователя задан надёжный пароль.
- [ ] Роль **Administrator** выдана только тем, кому она действительно нужна.
- [ ] У пользователей указан корректный часовой пояс.
- [ ] Неиспользуемые или скомпрометированные `API key` заменены.
- [ ] Удаление пользователей выполняется только после проверки последствий.

## Рекомендуемый порядок выдачи доступа

1. Создать пользователя.
2. Назначить роль.
3. Проверить профиль и часовой пояс.
4. При необходимости сформировать `API key`.
5. Установить пароль.
6. Проверить, что пользователь видит нужную стартовую страницу.
